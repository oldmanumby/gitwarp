## Challenges

### [Low] Challenge 1: Asset Path Mismatch for Favicon (`vite.svg` vs `favicon.svg`)

- **Assumption challenged**: The HTML entry point (`index.html`) correctly links to all static assets in `public/`.
- **Attack scenario**: When the user opens the web application in a browser, the browser attempts to fetch `/vite.svg` as specified in `<link rel="icon" type="image/svg+xml" href="/vite.svg" />`. Since `/vite.svg` does not exist in `public/` (the project includes `public/favicon.svg` and `public/icons.svg`), the request results in a 404 HTTP error.
- **Blast radius**: Cosmetic standard browser error log (404 for favicon); website functionality remains unaffected.
- **Mitigation**: Update line 5 of `index.html` from `<link rel="icon" type="image/svg+xml" href="/vite.svg" />` to `<link rel="icon" type="image/svg+xml" href="/favicon.svg" />`.
